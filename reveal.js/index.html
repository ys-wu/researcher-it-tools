Intro & Motivation

Process of Conducting Scientific Research

  • Identifying a Research Problem
  • Conducting a Literature Review
  • Formulating Hypotheses or Research Questions
  • Designing the Research Methodology
  • Collecting Data
  • Analyzing the Data
  • Drawing Interpretations and Conclusions
  • Documenting and Reporting Findings
  • Undergoing Peer Review

Purpose of Software Engineering

is to apply systematic, disciplined, and quantifiable approaches, to the development, operation, and maintenance of software systems.

Benefits of Software Engineering

  • Efficiency: Helps to optimize the use of resources, making software development more cost-effective and time-efficient.
  • Quality: By adhering to software engineering principles, developers can create software that is reliable, maintainable, and performs well.
  • Scalability: Software engineering practices facilitate the design of software that can easily scale to accommodate growth.
  • Manageability: Structured methodologies in software engineering allow for better management and control of software development projects.
  • Collaboration: Software engineering encourages teamwork and clear communication among team members, fostering an environment for collaborative problem-solving.
  • Reusability: One of the core principles of software engineering is designing for reusability, enabling the reuse of software components across different projects, thereby saving time and effort.

Frontend & Backend

Frontend

Also known as client-side programming, is the part of an application that users interact with directly.

Backend

Also referred to as server-side programming, is the part of the application that users don't see.

Frontend: HTML, CSS, JS

HTML: HyperText Markup Language

The standard markup language used to create the structure of web pages.

CSS: Cascading Style Sheets

A style-sheet language used for describing the look and formatting of a HTML document.

JS: JavaScript

A high-level, interpreted programming language used to enhance web pages with interactive features.

Database & Data Lake

Database

  • A structured set of data stored electronically.
  • Uses a schema to define how data is organized.
  • Optimized for data retrieval and updating operations.

Data Lake

  • A large storage repository of data in native format.
  • No schema, flexibility in storing diverse data.
  • Optimized for storing and analysising big data.
